# SuppCheck v1.1.0 安卓测试指南

## 🚀 快速开始

### 1. 确保虚拟机已启动
```bash
# 检查连接的设备
flutter devices

# 应该能看到类似输出：
# Android SDK built for x86 (mobile) • emulator-5554 • android-x86 • Android 13 (API 33)
```

### 2. 安装并运行
```bash
cd E:\SLU\code\suppcheck

# 调试模式运行（推荐测试）
flutter run -d android

# 或者构建 APK 后安装
flutter build apk --release
adb install build\app\outputs\flutter-apk\app-release.apk
```

## 🧪 测试清单

### ✅ 基础功能测试

| 功能 | 测试步骤 | 预期结果 |
|------|---------|---------|
| 添加补剂 | 点击"+" → 填写信息 → 选择分类 → 保存 | 补剂出现在列表中，带分类标签 |
| 今日打卡 | 点击"打卡服用" → 确认 | 进度条更新，今日记录显示新记录 |
| 撤销记录 | 今日记录列表 → 左滑 → 撤销 | 记录删除，库存恢复 |
| 连续打卡 | 连续多天打卡后查看首页 | 显示火焰图标和天数 |

### ✅ 新功能重点测试

#### 1. 补剂分类 🏷️
```
测试步骤：
1. 添加补剂时选择不同分类（维生素/蛋白质/益生菌等）
2. 保存后在今日页面和补剂列表查看
3. 确认每个分类显示对应的颜色标签
```

#### 2. 数据备份 💾
```
测试步骤：
1. 进入「设置」页面（首页右上角齿轮图标）
2. 点击「导出备份」→ 选择分享方式
3. 点击「复制到剪贴板」→ 粘贴到备忘录确认数据完整
4. 清除数据或卸载重装
5. 进入「设置」→「导入数据」→ 粘贴之前复制的JSON
6. 确认数据恢复完整
```

#### 3. 提醒管理 🔔
```
测试步骤：
1. 点击底部「提醒」导航
2. 查看所有补剂的提醒列表
3. 测试开关提醒
4. 点击「测试通知」确认通知权限和显示正常
```

#### 4. 日历历史记录 📅
```
测试步骤：
1. 连续几天打卡不同补剂
2. 进入「日历」页面
3. 点击不同日期查看历史记录
4. 确认今日显示进度条，历史日期显示状态
```

#### 5. 连续打卡统计 🔥
```
测试步骤：
1. 连续多天打卡（可以修改系统日期模拟）
2. 查看首页顶部的火焰图标
3. 统计页面也显示连续天数
```

## 🔧 常见问题排查

### 问题 1: 虚拟机连接不上
```bash
# 检查 ADB
cd %LOCALAPPDATA%\Android\Sdk\platform-tools
adb devices

# 如果显示 offline，重启 ADB
adb kill-server
adb start-server
```

### 问题 2: 构建失败
```bash
# 清理并重建
flutter clean
flutter pub get
flutter run
```

### 问题 3: 通知不显示
```bash
# 检查权限
adb shell appops query-ops com.example.suppcheck

# 或在虚拟机设置中手动开启通知权限
设置 → 应用 → SuppCheck → 通知 → 允许
```
